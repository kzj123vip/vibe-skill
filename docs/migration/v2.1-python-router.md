# v2.1.0 迁移指南：Python 优先路由器

## 概述

从 v2.1.0 开始，Vibe 默认使用 **Python 路由器**，消除了 macOS/Linux 用户对 PowerShell 的依赖。

## 变更摘要

### 之前（v2.0.x）

```python
# router_bridge.py 第 75 行
shell = None if args.force_runtime_neutral else resolve_powershell_host()

if shell:
    payload = invoke_canonical_router(args, shell)  # 优先 PowerShell
else:
    payload = route_prompt(...)  # 降级到 Python
```

**行为**：如果系统安装了 PowerShell → 自动使用 PowerShell 路由器

### 之后（v2.1.0+）

```python
# router_bridge.py 新逻辑
use_powershell = (
    not args.force_runtime_neutral
    and os.environ.get('VIBE_USE_POWERSHELL', '').strip() in ('1', 'true', 'True', 'TRUE')
)

shell = resolve_powershell_host() if use_powershell else None

if shell:
    payload = invoke_canonical_router(args, shell)  # PowerShell 回退（需环境变量）
else:
    payload = route_prompt(...)  # 默认 Python 路由器
```

**行为**：默认使用 Python 路由器，除非设置 `VIBE_USE_POWERSHELL=1`

## 影响分析

### ✅ 无影响用户

- **macOS/Linux 用户**：体验改进，无需安装 PowerShell
- **使用 `--force-runtime-neutral` 的用户**：行为不变，仍强制 Python
- **未安装 PowerShell 的用户**：行为不变，本来就用 Python

### ⚠️ 可能受影响用户

- **Windows 用户且依赖 PowerShell 特性**：需要设置环境变量 `VIBE_USE_POWERSHELL=1`
- **CI/CD 显式调用 PowerShell 路由器**：需要更新脚本

## 迁移步骤

### 场景 1：macOS/Linux 用户（推荐，无操作）

**之前**：系统有 PowerShell → 自动使用 PowerShell  
**之后**：系统有 Python → 自动使用 Python（更快）

✅ **无需任何操作**

### 场景 2：Windows 用户（希望继续使用 PowerShell）

**步骤**：

1. 设置用户级环境变量（一次性）：

```powershell
# PowerShell（管理员）
[System.Environment]::SetEnvironmentVariable('VIBE_USE_POWERSHELL', '1', 'User')
```

2. 或在脚本中临时设置：

```powershell
$env:VIBE_USE_POWERSHELL=1
python3 -m vgo_runtime.router_bridge --prompt "任务" --grade M --task-type coding
```

### 场景 3：CI/CD 脚本

**如果之前依赖 PowerShell 路由器**：

```yaml
# GitHub Actions 示例
- name: 设置 PowerShell 路由器
  run: echo "VIBE_USE_POWERSHELL=1" >> $GITHUB_ENV

- name: 运行路由器
  run: python3 -m vgo_runtime.router_bridge --prompt "$TASK" --grade M --task-type coding
```

## 验证

### 验证当前使用的路由器

```bash
# 方法 1：检查输出 JSON 中的 runtime_neutral_bridge 字段
python3 -m vgo_runtime.router_bridge --prompt "测试" --grade M --task-type coding | \
  python3 -c "import json, sys; data=json.load(sys.stdin); print(data.get('runtime_neutral_bridge', {}))"

# 预期输出（Python）：
# {'enabled': True, 'engine': 'python', 'host': 'runtime_neutral'}

# 预期输出（PowerShell）：
# {} 或字段不存在
```

### 验证 PowerShell 回退

```bash

python3 -m vgo_runtime.router_bridge --prompt "测试" --grade M --task-type coding

# 应该调用 PowerShell 脚本（输出格式略有不同）
```

## 技术细节

### Python 路由器优势

1. **依赖更少**：不需要安装 PowerShell（macOS/Linux 常见场景）
2. **启动更快**：无需启动外部进程（PowerShell）
3. **维护性更好**：Python 代码库更易维护和扩展
4. **功能完整**：Python 路由器已包含所有核心功能（823 行）

### PowerShell 路由器保留原因

1. **向后兼容**：现有 Windows 用户可能依赖 PowerShell 特性
2. **功能丰富**：PowerShell 脚本包含 1910 行逻辑（部分历史特性）
3. **测试覆盖**：保留作为回归测试的参考实现

## FAQ

**Q: 为什么要改为 Python 优先？**  
A: macOS 用户反馈需要安装 PowerShell 才能使用，增加了使用门槛。Python 是核心依赖，无需额外安装。

**Q: PowerShell 路由器会被移除吗？**  
A: 短期内不会。保留作为兼容回退路径，直到确认所有用户迁移完成。

**Q: Python 路由器功能完整吗？**  
A: 是的。Python 路由器（823 行）包含所有核心功能：specialist 路由、权威决策、深度发现、降级逻辑等。

**Q: 如何确认我用的是哪个路由器？**  
A: 检查输出 JSON 的 `runtime_neutral_bridge.engine` 字段，值为 `python` 表示 Python 路由器。
